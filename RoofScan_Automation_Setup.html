Complete Automation Setup Guide — 16 Zaps

Before you build a single Zap
Connect every tool to Zapier first. Do this in one sitting — it takes about 20 minutes. Every automation you build after this will pull from these connections. Do not skip any.
⚡
Zapier account
Go to zapier.com → sign up with info@roofscanuk.co.uk. Choose the Starter plan (free tier: 100 tasks/month — enough for launch). Keep this tab open throughout the setup.
✉️
Gmail (Google Workspace)
In Zapier → My Apps → Add connection → search Gmail → sign in with info@roofscanuk.co.uk → allow all permissions. Every automated email sends from this address.
📅
Cal.com
In Zapier → My Apps → search Cal.com → connect → sign in with info@roofscanuk.co.uk → authorise. Zapier will see both your inspection and subscription-call events.
🗄️
Airtable
In Zapier → My Apps → search Airtable → connect → sign in with info@roofscanuk.co.uk → authorise access to all bases. Your RoofScan UK Jobs base must already exist.
💳
Stripe
In Zapier → My Apps → search Stripe → connect → sign in to your Stripe account → authorise. Use your live account, not test mode, once you are ready to take real payments.
🏦
GoCardless
In Zapier → My Apps → search GoCardless → connect → sign in → authorise. GoCardless connection is needed for the failed payment alert automation.
📆
Google Calendar
In Zapier → My Apps → search Google Calendar → connect → sign in with info@roofscanuk.co.uk (same Workspace account as Gmail) → authorise. This is for the pre-job briefing automation.
⚠ Webhook URL
Automation 12 (quote capture) uses a Zapier webhook. When you build that Zap you will get a unique webhook URL. Copy it and paste it into two places in your website code: QuoteCalculator.jsx and BookingFlow.jsx where it says YOUR_ZAPIER_QUOTE_WEBHOOK_URL. Your developer can do this when building the site.
Set up Airtable before building any Zaps
You need four tables. The Zaps write to and read from these. Get the structure right first or you will have to redo the Zaps later.
🗄️
Jobs table — Your main jobs CRM — one record per inspection
Create in Airtable
Base: RoofScan UK Jobs → + Add table → name it "Jobs" → add these fields:
+
Job Reference (text) — e.g. RSU-0001. Fill in manually when creating.
+
Client Name (text) — Full name. Zapier fills this from Cal.com booking.
+
Property Address (text) — Zapier fills from Cal.com pre-booking question 1.
+
Client Email (email) — Zapier fills from Cal.com.
+
Client Phone (phone) — Zapier fills from Cal.com.
+
Property Type (single select) — Options: Terrace / Semi-detached / Detached / Large detached. Zapier fills from Cal.com question 2.
+
Quote Total (currency (£)) — Zapier fills from Cal.com URL parameter.
+
Deposit Paid (checkbox) — Check manually once Stripe confirms deposit.
+
Inspection Date (date) — Zapier fills from Cal.com booking date.
+
Status (single select) — Options: Booked / Complete / Paid / Report Sent. You update this manually after each stage.
+
Balance Due (formula) — Formula: {Quote Total} - 50
+
Notes (long text) — Access notes, anything unusual. Zapier fills from Cal.com question 3.
+
Report Link (URL) — Paste the Google Drive report link once written.
+
Subscription Price (currency (£)) — For sub clients only — the agreed per-job rate (149/165/189/225).
+
Job Type (single select) — Options: One-off / Subscription
+
Repair Required (checkbox) — Tick this to trigger the roofer referral automation.

The golden rule — one message, one sender
Before you build a single Zap, configure these settings in each tool. This prevents clients receiving the same email twice from different systems. Do this before testing anything.
Cal.com
Booking confirmation email to clientLeave ON — Cal.com is the sole sender
24-hour reminder email to clientLeave ON — Cal.com is the sole sender
Turn OFF any Zapier booking emails to clientYou must NOT build a Zap that also sends a booking confirmation — Cal.com already does it
No-show optionTurn ON in each event → Workflows → add no-show workflow → sends rescheduling link automatically
Stripe
Automatic receipt email (deposit)Leave ON — Stripe sends this automatically
Automatic receipt email (balance)Leave ON — Stripe sends this automatically
Invoice emails (Stripe-generated)Turn OFF — Zapier owns the balance payment REQUEST email. Go to Settings → Billing → Invoice emails → disable
Where to turn it offStripe Dashboard → Settings → Customer emails → uncheck invoice/payment emails
Xero
Subscription invoice emailsXero sends these — do NOT duplicate in Zapier
Invoice reminder emailsTurn ON in Xero → Settings → Invoice reminders → set at 7 days and 14 days overdue
Where to turn on remindersXero → Settings → Invoice reminders → turn on → set 7 days overdue and 14 days overdue
Payment syncStripe, GoCardless and Starling feed Xero automatically — do not create manual records
Zapier (what you own)
Balance payment requestZapier ONLY — do not let Stripe also send an invoice
Report delivery emailZapier ONLY — triggered by Airtable status change
Review requestZapier ONLY — 24hrs after payment
Annual re-inspection reminderZapier ONLY — 11 months after paid
All internal alerts to TakZapier ONLY — never goes to client
Automations 1–4: The core job flow
These four run on every single job. Build and test them in order — each one depends on the previous stage completing correctly.
📋
Zap 1: New booking → Airtable job record + notify Tak
Zapier

Every time a client books via Cal.com, a job record is created in Airtable automatically and Tak gets a notification. Cal.com sends the client confirmation separately — do NOT add a client email here.

TRIGGER → Cal.com: Event: Booking Created
Trigger step → search Cal.com → choose 'Booking Created' → connect your Cal.com account → select the Drone Roof Inspection event → click Test trigger → a sample booking should appear. If no test data, create a test booking on your Cal.com link first.
ACTION 1 → Airtable: Create Record
Add action → search Airtable → choose 'Create Record' → select your RoofScan UK Jobs base → select the Jobs table → map these fields:
• Client Name → Booking: Attendee Name
• Client Email → Booking: Attendee Email
• Client Phone → Booking: Attendee Phone
• Inspection Date → Booking: Start Time
• Property Address → Booking: Answer to Question 1 (the address question)
• Property Type → Booking: Answer to Question 2
• Notes → Booking: Answer to Question 3 (access notes)
• Quote Total → Booking: Custom Parameter 'total' from the URL
• Status → type 'Booked' (static text)
• Job Type → type 'One-off' (static text)
ACTION 2 → Gmail: Send Email
Add action → Gmail → Send Email → fill in:
• To: info@roofscanuk.co.uk (your own email — this is internal only)
• Subject: New booking — [Client Name] — [Property Address]
• Body: New job booked. Client: [Name]. Address: [Address]. Date: [Inspection Date]. Quote: £[Quote Total]. Check Airtable for full details.
✓ How to test
Create a real test booking via cal.com/roofscanuk/inspection and confirm the Airtable record is created with all fields populated. Check your email for the notification.

Zap 2: Job Complete → balance payment request to client
Zapier

When you mark a job as Complete in Airtable, Zapier sends the client their balance payment link. This is the ONLY system that sends this email — Stripe must not send a duplicate.

TRIGGER → Airtable: Event: Record Updated
Add trigger → Airtable → Record Updated → select Jobs base → select Jobs table → Field to monitor: Status → click Test trigger
FILTER → Zapier: Filter: Only continue if Status = Complete
Add a Filter step → Only continue if → Status (Text) Exactly matches → Complete. This prevents the Zap firing on every status change.
ACTION 1 → Stripe: Create Payment Link

Body: Hi [Client Name], your roof inspection at [address] is now complete. Your report is being prepared and will be with you within 48 hours.

[Payment Link from previous action]

Zap 3: Balance paid → mark Paid + review request
Zapier

When Stripe confirms a balance payment, Airtable is updated to Paid and a review request is sent 24 hours later.

TRIGGER → Stripe: Event: Payment Intent Succeeded
Trigger → Stripe → Payment Intent Succeeded → Test with a real or test payment
FILTER → Zapier: Filter: Amount > 5000 (pence)
Filter to exclude the £50 deposit — only fire on full balance payments. Amount greater than 5000 pence (= £50). Adjust if your smallest balance is more than £50.

Zap 17: Subscription intake form → notify Tak
Zapier

When a subscription client submits a property via the Airtable intake form, a job record is created but you are never told. This Zap fixes that — you get an immediate notification every time a subscription property lands so you can schedule it.

TRIGGER → Airtable: New Record in Jobs table
Airtable → New Record → RoofScan UK Jobs base → Jobs table. This fires whenever any new record is created — we use a filter to catch only subscription jobs.
FILTER → Job Type = Subscription
Filter → Job Type (Text) Exactly matches → Subscription. This ensures Cal.com one-off bookings (handled by Zap 1) do not also trigger this notification.
ACTION → Gmail: Notify Tak

Log into Airtable to schedule this job.

Zap 18: Cal.com booking → mark Quotes record as Converted
Zapier

This is the most important of the three. Without it, someone who gets a quote on Monday and books on Tuesday still receives the 3-day follow-up on Thursday — which looks unprofessional. This Zap updates their Quotes record to Converted the moment they book so the follow-up is suppressed.

Build this alongside Zap 1
This Zap uses the same Cal.com trigger as Zap 1. You can add it as an extra action inside Zap 1 (recommended — fewer Zaps to manage) or build it as a separate Zap. Instructions below are for adding it to Zap 1.
OPEN ZAP 1 → Add action after the existing actions
In Zapier → open Zap 1 (New booking → Airtable record) → scroll to the bottom → Add action
NEW ACTION 3 → Airtable: Find Record in Quotes table
Airtable → Find Record → RoofScan UK Jobs base → Quotes table → Search field: Email → Search value: [Attendee Email from Cal.com]

This looks for a Quotes record with the same email as the person who just booked. If they got a quote earlier, it will find it.
NEW ACTION 4 → Airtable: Update Record (only if found)
Airtable → Update Record → Quotes table → Record ID: [ID from the Find Record step]

Field to update: Status → Value: Converted

Important: tick "Only continue if previous step found a record" in the filter settings — otherwise this action errors when someone books without having got a quote first.

Zap 19: Subscription job scheduled → create Google Calendar event
Zapier

Subscription jobs come in via the Airtable intake form — not Cal.com — so no calendar event is ever created. Without one, Zap 7 (the pre-job briefing) never fires. This Zap creates the Google Calendar event the moment you set an Inspection Date on a subscription job, so the briefing email arrives the night before just like it does for one-off jobs.

TRIGGER → Airtable: Record Updated — Jobs table
Airtable → Record Updated → Jobs table → monitor the Inspection Date field. This fires whenever an Inspection Date is added or changed.
FILTER → Job Type = Subscription AND Inspection Date is not empty
Filter → Job Type exactly matches Subscription → AND → Inspection Date exists (is not empty). This prevents one-off jobs (which already have a Cal.com event) from creating a duplicate calendar entry.
ACTION → Google Calendar: Create Event
Google Calendar → Create Detailed Event → fill in:
Calendar: Jobs (your jobs calendar)
Title: [Property Address] — [Client Name] — Subscription
Start date/time: [Inspection Date from Airtable]
End date/time: [Inspection Date + 2 hours]
Description: Client: [Client Name] · Phone: [Client Phone] · Type: [Property Type] · Access: [Notes] · Quote: £[Subscription Price]

This event appears in your Jobs calendar. Zap 7 then picks it up the evening before and sends your pre-job briefing automatically.
✓ How to test
Set an Inspection Date on a subscription job record in Airtable. Check your Jobs calendar — an event should appear within a few minutes. The night before that date, confirm Zap 7 fires and sends your briefing email.
